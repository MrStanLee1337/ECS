Проект демонстрирует реализацию ECS для игрового движка (сущности, компоненты, системы)
с возможностью управлениями ими с помощью скриптов на Lua.

Архитектура

Registry - создание сущностей, управление пулом(Pool) и системами.

Pool - хранилище компонентов.

Entity - uint32_t, уникальный ID для объектов.

System - базовый класс для игровой логики(update).

Компоненты хранятся как unique_ptr<>, что гарантирует автоматическое управление памятью.
Lua скрипты могут считывать и изменять компоненты сущностей без использования глобальных переменных.

Сборка
Компилятор С++20 и выше, Lua 5.5.0, CMAKE

Откройте папку с файлами в консоли:

mkdir build 

cd build

cmake .. 

cmake --build .

cd Debug

NauEngineTest.exe
